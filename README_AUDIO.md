## ⚙️ 配置参数

### 频谱效果参数（`effects-init.js` 第 32 行）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `fftSize` | 256 | FFT 大小，控制频率分辨率 |
| `barCount` | 64 (桌面) / 32 (移动) | 频谱柱数量 |
| `barColor` | rgba(100, 220, 220, 0.75) | 主柱子颜色 |
| `barGlowColor` | rgba(255, 255, 255, 0.5) | 光晕颜色 |
| `position` | 'bottom' | 位置（bottom/top/center） |
| `sensitivity` | 1.3 | 灵敏度倍数 |
| `opacity` | 0.8 | 整体透明度 |
| `smoothingTimeConstant` | 0.78 | 平滑系数 |

### 音频控制参数（`effects-init.js` 第 44 行）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `position` | 'bottom-right' | 控制面板位置 |
| `defaultVolume` | 0.5 | 默认音量（0.0-1.0） |
| `autoPlay` | false | 是否自动播放 |

## 🔍 技术细节

### Web Audio API 流程

```
音频源 (Audio Element / MediaStream)
    ↓
MediaElementSource / MediaStreamSource
    ↓
AnalyserNode (FFT 分析)
    ↓
getByteFrequencyData() → Uint8Array
    ↓
Canvas 绘制 (柱状图)
    ↓
AudioContext.destination (扬声器输出)
```

### 频谱计算

1. **FFT 分析**：将时域信号转换为频域
2. **降采样**：多个频率样本求平均 → 一个柱子
3. **归一化**：0-255 映射到 0.0-1.0
4. **灵敏度调整**：乘以 `sensitivity` 倍数
5. **高度计算**：`normalizedHeight × maxBarHeight`

## 🐛 故障排查

### 问题 1：频谱不动
**原因**：音频未播放  
**解决**：点击播放按钮，检查音量不为 0

### 问题 2：找不到音频文件
**原因**：文件路径或名称错误  
**解决**：确认 `assets/background-music.mp3` 存在

### 问题 3：浏览器阻止自动播放
**原因**：现代浏览器自动播放策略  
**解决**：需要用户交互（已设置 `autoPlay: false`）

### 问题 4：CORS 错误（CDN 音频）
**原因**：跨域资源未允许  
**解决**：确保音频服务器设置 CORS 头

### 问题 5：性能卡顿
**原因**：参数过高或设备性能不足  
**解决**：
- 减少 `fftSize`: 256 → 128
- 减少 `barCount`: 64 → 32
- 增加平滑度: 0.78 → 0.85

## 🎵 推荐音频特征

### 适合频谱可视化的音乐类型：
- ✅ 电子音乐（EDM）- 节奏明显
- ✅ Hip-Hop/Trap - 低频丰富
- ✅ 摇滚/金属 - 能量强
- ✅ 鼓点音乐 - 冲击力强

### 不太适合的类型：
- ⚠️ 古典音乐 - 动态范围大，需调整灵敏度
- ⚠️ 环境音/白噪音 - 频谱变化小
